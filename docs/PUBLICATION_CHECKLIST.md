# Publication checklist

This file records release blockers that cannot be solved safely by deleting old files.

## Completed repository hygiene

- Core source, reproducible maintenance scripts, tests, and the canonical ProtOS
  `type_I_opsins.csv`
  table are separated from runtime data and generated outputs.
- Generated HTML, run logs, cached analyses, and historical clustered-MO artifacts are
  not tracked.
- Runtime and development dependencies are declared separately.
- Citation metadata and a single current README are present.
- Tandem TARA preprocessing and manual register corrections are encoded and tested.

## Required before archival release

- [ ] Select and add an explicit software license.
- [ ] Publish an immutable Zenodo data record and replace `<ZENODO_RECORD_ID>` in the
      README with its record identifier and DOI.
- [x] Freeze the canonical publication metadata input as `mo_exp_ST5_HEK1.xlsx`;
      `build_groundtruth_bundle.py` normalizes it to `property.csv`.
- [x] Pin and test the pulled ProtOS `master` revision and its 130-row
      `type_I_opsins.csv` table from a clean runtime initialization.
- [x] Run the complete preparation and analysis pipeline without caches.
- [ ] Resolve every cell-level reference/structure conflict reported by
      `build_groundtruth_bundle.py`; the builder must not emit a ZIP while any remain.
- [ ] Record checksums for the input metadata, canonical GRN table, structure archive,
      primary tabular outputs, and final figures.
- [ ] Confirm that every manuscript figure and table is generated by a retained command
      or is included in the immutable data archive with provenance.
- [ ] Add the manuscript DOI and full author list to `CITATION.cff` when available.
- [ ] Create a version tag only after the clean-room reproduction succeeds.

## Release evidence to retain

- Python and operating-system versions;
- exact MOGRN and ProtOS commits;
- Zenodo record DOI and archive checksums;
- commands used for preparation, analysis, reference synchronization, and plotting;
- test output and a concise list of accepted manual GRN curations.
