## プロジェクト概要

MOGRN（Microbial Opsin Generic Residue Numbering）プロジェクトは、微生物オプシン構造を解析するための包括的なフレームワークを提供します。ワークフローは現在6つの主要ステップを含みます：

1. **データ準備**: インフラストラクチャの初期化とデータセットの検証
2. **配列設定**: タンパク質配列のYAMLファイル生成
3. **コア解析**: 構造アラインメントとGRN割り当ての実行
4. **可視化**: 標準的なプロットとヒートマップの作成
5. **保存性解析**: GRN保存パターンの解析
6. **モチーフ解析**: 機能的モチーフの検出と検証

### 汎用残基番号付け（GRN）とは？

GRNは、異なるオプシン構造間で残基位置を比較するための標準化された方法を提供します。キーポジションは「.50」で終わり（例：3.50、7.50）、構造的アンカーとして機能します。これにより、タンパク質の長さや挿入/欠失が異なる場合でも一貫した比較が可能になります。

## 前提条件

### システム要件
- Python 3.10以上
- 理想的には16GB以上のRAM

### 依存関係のインストール

1. **conda環境の作成と有効化**（推奨）：
```bash
conda create -n mogrn python=3.10
conda activate mogrn
```

2. **Pythonパッケージのインストール**：
```bash
pip install -r requirements.txt
```

3. **Protosフレームワークのインストール**：
```bash
git clone https://github.com/flurinh/protos.git
cd protos
pip install -e .
cd ..
```

### 必要なデータファイル

以下のディレクトリを作成し、データファイルを追加してください：

```
MOGRN/
├── property/
│   ├── mo_exp.csv        # 必須：実験的プロパティ
│   └── helices.json      # 必須：ヘリックス定義
└── structures/
    ├── hideaki_exp/      # 実験構造（CIF形式）
    ├── hideaki_pred/     # 予測構造（CIF形式）
    └── mo_pred/          # 追加の予測（CIF形式）
```

## ステップバイステップ解析

### ステップ1：データ準備

**コマンド：**
```bash
python prepare_data.py
```

**処理内容：**
- ファイル管理のためのProtosPathsシステムの初期化
- 必要なディレクトリ構造の作成
- 構造処理のためのCifBaseProcessorのセットアップ
- すべての構造ファイルへのアクセス可能性の検証
- 効率的な処理のためのキャッシュディレクトリの作成

**期待される出力：**
```
データディレクトリを設定しています...
ProtosPathsを初期化しています...
データセットを処理しています：
  - mo_exp: X個の構造
  - mo_pred: Y個の構造
  - hideaki_exp: Z個の構造
  - hideaki_pred: W個の構造
データ準備が完了しました！
```

**トラブルシューティング：**
- 「FileNotFoundError」が表示される場合は、property/とstructures/ディレクトリが存在することを確認してください
- 構造が見つからない場合は、CIFファイルが正しいサブディレクトリにあることを確認してください

### ステップ2：配列設定

**コマンド：**
```bash
python prepare_yaml.py
```

**処理内容：**
- property/mo_exp.csvからタンパク質配列を読み込み
- 各タンパク質の個別YAMLファイルを作成
- 特殊なケース（複数のレチナール、長い配列）の処理
- 配列の完全性を検証

**期待される出力：**
```
property/mo_exp.csvから配列を処理しています...
X個のタンパク質のYAMLファイルを作成しました
特殊なケース：
  - 複数のレチナールを持つタンパク質：Y個
  - 長い配列（400aa以上）：Z個
YAML準備が完了しました！
```

### ステップ3：コア解析ワークフロー

**コマンド：**
```bash
python opsin_analysis_workflow.py
```

**処理内容：**
1. **構造読み込み**: キャッシング付きですべての構造を読み込み
2. **RMSD計算**: ペアワイズ構造類似性を計算
3. **ヘリックス割り当て**: 膜貫通領域を識別
4. **GRN割り当て**: 標準化された番号付けで多重配列アラインメントを実行
5. **距離解析**: レチナールへの残基距離を計算

### ステップ4：標準的な可視化

**コマンド：**
```bash
python plot.py
```

**処理内容：**
- output ディレクトリから解析結果を読み込み
- 様々な可視化タイプを作成
- 品質フィルターを適用
- 出版可能な図を生成

**主要な可視化：**
- `rmsd_heatmap.png`: 構造類似性行列
- `similarity_tree.png`: 階層的クラスタリング
- `conservation_plots/`: 位置特異的保存性
- `helix_logos/`: 各ヘリックスの配列ロゴ

### ステップ5：GRN保存性解析（新機能）

**コマンド：**
```bash
python analyze_grns.py
```

**処理内容：**
1. **保存性解析**: 機能グループ別に.50位置での保存性を調査
2. **パターン検出**: ポンプとチャネルを区別する位置を特定
3. **共進化解析**: 相関する残基変化を検出
4. **レポート生成**: 詳細な解釈レポートを作成

**確認すべき主要結果：**
1. **包括的サマリー図**: すべての主要な発見を示すマルチパネル可視化
2. **識別ヒートマップ**: どの位置が機能グループを区別するかを表示
3. **共進化パターン**: 相関する位置のネットワーク

### ステップ6：モチーフパターン解析（新機能）

**コマンド：**
```bash
python analyze_motifs.py
```

**処理内容：**
1. **単一位置解析**: 各GRN位置での保存性
2. **文献モチーフ検証**: 既知のモチーフ（DTDなど）をチェック
3. **相関解析**: 新しい相関パターンを発見
4. **新規モチーフ検出**: 以前に知られていないパターンを特定

**主要な出力：**
1. **拡張モチーフテーブル**: すべてのモチーフの統計付き包括的テーブル
2. **相関行列**: どの位置が共変するかを表示
3. **新規ファミリー検出**: 機能グループ特有の新しいモチーフパターン

## 結果の理解

### 保存スコア
- **90%以上**: 普遍的に保存され、機能に不可欠
- **70-90%**: 機能グループ内で高度に保存
- **50-70%**: 中程度に保存、特異性を決定する可能性
- **50%未満**: 可変位置、調節に関与する可能性

### 機能的識別
解析は機能グループを区別する「シグネチャー」位置を特定します：
- **位置3.50**: ポンプのT（スレオニン）→チャネルのC（システイン）
- **位置5.50**: ポンプのS（セリン）→チャネルのG（グリシン）
- **位置4.50**: ポンプのM（メチオニン）→チャネルのT/R

### モチーフの解釈
- **保存されたモチーフ**: 基本的なオプシン機能に不可欠
- **修飾されたモチーフ**: 特定の機能特性を決定
- **新規モチーフ**: 未発見の機能要素を表す可能性

## トラブルシューティング

### よくある問題と解決策

1. **ModuleNotFoundError: No module named 'protos'**
   - 解決策: Protosがインストールされていることを確認: `cd protos && pip install -e .`

2. **RMSD計算中のMemoryError**
   - 解決策: `--no-cache`フラグを使用

3. **空の図または欠損データ**
   - 前のステップが正常に完了したことを確認
   - 期待されるディレクトリに出力ファイルが存在することを確認
   - mo_exp.csvのタンパク質プロパティマッピングを確認

4. **パフォーマンスが遅い**
   - キャッシングを有効にする（デフォルト）
   - より良いPCを入手する ;)
