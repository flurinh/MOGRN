# MOGRNプロジェクトガイド

このガイドでは、微生物オプシン解析ワークフローの実行方法と各スクリプトの役割について説明します。

## プロジェクト概要

オプシン解析プロジェクトは、汎用残基番号付け（GRN）を使用して実験的および予測的オプシン構造を分析、比較、可視化するための包括的なフレームワークを提供します。このプロジェクトは、様々な感覚機能に関与する光感受性タンパク質である微生物オプシンの構造解析に焦点を当てています。

分析パイプラインは以下の複数のコンポーネントを統合しています：
- 様々なソースからの構造データの読み込みと前処理
- 構造的アラインメントとエラー計算
- 膜貫通ヘリックスの識別とアノテーション
- 複数の構造間の比較分析
- 標準化された比較のための汎用残基番号（GRN）の割り当て
- 構造的洞察のための包括的な可視化ツール

## モジュールアーキテクチャ

ワークフローは専門化されたモジュールに分かれています：

- **data_processing.py**: オプシン構造の読み込み、フィルタリング、前処理
- **structure_comparison.py**: 構造の比較とアラインメント、RMSDの計算
- **helix_analysis.py**: 膜貫通ヘリックスの識別とアノテーション
- **error_analysis.py**: 実験構造と予測構造間のエラー分析
- **msa_grn.py**: 多重配列アラインメントとGRN割り当ての処理
- **visualization_functions.py**: アラインメントと分析結果の可視化の作成
- **foldmason_helpers.py**: FoldMason構造アラインメントツールとのインターフェース
- **opsin_analysis_workflow.py**: 完全な分析パイプラインの調整

## ステップ1: データインフラの準備とデータセットの初期化

**実行コマンド:**
```bash
# データ構造のセットアップとデータセットの処理
python prepare_data_fixed.py
```

**このステップの内容:**
- CSV ファイルに保存されている微生物オプシンのプロパティデータの確認と整理
- 分析に必要なディレクトリ構造の設定
- ファイルパスを管理するための Protos フレームワークから ProtosPaths システムの初期化
- ユーザー書き込み可能データと参照データのデータディレクトリの作成
- ディレクトリ構造の検証と問題の報告
- タンパク質構造を管理するための CifBaseProcessor のセットアップ
- レジストリファイルの作成とデータセット管理の初期化
- 4種類のデータセットの処理:
  1. 実験的微生物オプシン構造（PDBからダウンロード）
  2. 予測的微生物オプシン構造（ローカルファイルから）
  3. Hideakiデータセットからの実験構造
  4. Hideakiデータセットからの予測構造
- すべての構造にアクセスできることを確認するためのデータセット検証
- 効率的なデータ取得のためのキャッシングのセットアップ

## ステップ2: タンパク質配列のYAML設定ファイルの作成

**実行コマンド:**
```bash
# タンパク質配列からYAMLファイルを生成
python prepare_yaml.py
```

**このステップの内容:**
- CSVファイルからのタンパク質配列データの処理
- 適切な配列フォーマットを持つ各タンパク質のYAMLファイルの作成
- 利用可能な場合、開始/終了位置を使用した配列セグメントの抽出
- 範囲情報のない400アミノ酸を超える配列に関するレポート
- 複数のレチナールを持つタンパク質の特別な設定の作成
- アミノ酸配列の検証と無効な文字のクリーニング
- 配列長分布の統計と可視化の生成

## ステップ3: 分析ワークフローの実行

**実行コマンド:**
```bash
# 完全な分析パイプラインを実行
python opsin_analysis_workflow.py
```

**このステップの内容:**
- ステップ1で作成されたデータセットからすべての構造を読み込み
- 構造的アラインメントと標準化の実行
- 実験構造と予測構造間の構造的エラーの計算
- 膜貫通ヘリカル領域の識別とアノテーション
- 類似点と相違点を分析するための構造比較
- クロス比較のための統一構造マッピングの作成
- 残基位置を標準化するための汎用残基番号（GRN）の割り当て
- より速い再利用のための各段階での結果のキャッシング

このスクリプトは、標準ワークフローに適切に設定されたデフォルトパラメータを使用します。

ワークフローは以下の主要な処理ステップに従います：
1. **構造読み込み**: キャッシングサポート付きで準備されたデータセットからオプシン構造を読み込み
2. **エラー計算**: ペアになった実験構造と予測構造間のRMSDエラーを計算
3. **ヘリックスアノテーション**: 膜貫通指定でヘリカルセグメントを識別およびラベル付け
4. **構造比較**: RMSD行列を作成し、構造関係を可視化
5. **GRN割り当て**: 構造をアラインメントし、進化的保存に基づいて標準番号付けを割り当て

## ステップ4: 可視化の生成

**実行コマンド:**
```bash
# 標準化された可視化とプロットを作成
python plot_figures.py
```

**このステップの内容:**
- 分析ワークフローから事前計算されたデータを読み込み
- 複数の可視化タイプの生成:
  - 構造的類似性を示すRMSDヒートマップ
  - 類似性によって構造をクラスタリングする類似性ツリー
  - レチナールへの近接度を示す距離プロット
  - アミノ酸保存を示す保存プロット
  - 配列パターンを表示するヘリックスロゴプロット
  - 主要な発見をまとめる概要プロット
- 低品質構造のフィルタリング
- 主要なメトリクスを含むタンパク質概要CSVの作成
- すべての可視化を「figures」サブディレクトリに保存

このスクリプトは、デフォルトの出力ディレクトリ内の分析ワークフローによって作成されたデータを自動的に検出して使用します。

可視化には以下が含まれます：
- 構造類似性ヒートマップとデンドログラム
- レチナールへの近接度を示す残基距離プロット
- ヘリックスアノテーション可視化
- 残基保存分析
- 結合ポケット比較
- 構造的特徴の統計的要約

## 出力ファイル

分析では以下の主要な出力ファイルが生成されます：

- **`rmsd_matrix.csv`**: すべての構造間のペアワイズRMSD値
- **`msa_table_grn.csv`**: GRN位置を含む多重配列アラインメント
- **`ca_distance_table_grn.csv`**: 各残基からレチナールへのC-アルファ距離
- **`distance_table_grn.csv`**: GRN位置を含むレチナールへの側鎖距離
- **`protein_summary.csv`**: 主要なメトリクスを含む分析されたタンパク質の要約

## オプションの高度な使用法

デフォルト設定を超えてワークフローをカスタマイズする必要がある場合、スクリプトはさまざまなコマンドラインオプションをサポートしています。以下にいくつかの例を示します：

### カスタムディレクトリ
```bash
# カスタム出力ディレクトリを指定
python opsin_analysis_workflow.py --output-dir custom_output

# 可視化の入力と出力ディレクトリを指定
python plot_figures.py --input-dir custom_output --output-dir custom_figures
```

### キャッシュ制御
```bash
# キャッシュされた結果を使用せずに実行（すべてを再計算）
python opsin_analysis_workflow.py --no-cache
```

### 可視化制御
```bash
# 分析中に可視化を生成せずに実行
python opsin_analysis_workflow.py --no-visualize

# より高品質の図を生成（時間がかかります）
python plot_figures.py --quality high
```

これらの高度なオプションは、デフォルトの動作をオーバーライドする必要がある場合にのみ使用してください。
