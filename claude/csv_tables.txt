# CSV HANDLING RULES & BEST PRACTICES

## 🎯 PRIMARY GOALS
1. **UNDERSTAND TABLE CONTEXT FIRST** - What does this table represent?
2. **MINIMIZE TOKEN USAGE** - Read only what's necessary
3. **ENABLE PRECISE ACCESS** - Use indices and column names for targeted retrieval
4. **REQUIRES THE PYTHON ENV TO BE ACTIVATED**

## 📊 TABLE CONTEXT IN MOGRN PROJECT

### Known Table Types:
1. **RMSD Matrix** (`rmsd_matrix.csv`)
   - Structure: PDB IDs as both row and column indices
   - Content: RMSD values between structure pairs
   - Usage: Structure similarity analysis

2. **MSA with GRN** (`msa_table_grn.csv`)
   - Structure: PDB IDs as rows, GRN positions as columns
   - Content: Amino acid residues at each GRN position
   - Usage: Multiple sequence alignment analysis

3. **Distance Tables** (`ca_distance_table_grn.csv`, `distance_table_grn.csv`)
   - Structure: PDB IDs as rows, GRN positions as columns
   - Content: Distances from residues to retinal
   - Usage: Proximity analysis

4. **Protein Summary** (`protein_summary.csv`)
   - Structure: PDB IDs as rows, various metrics as columns
   - Content: Summary statistics per protein
   - Usage: Overview and filtering

5. **Property Data** (`mo_exp.csv`)
   - Structure: Protein IDs as rows, properties as columns
   - Content: Experimental properties and metadata
   - Usage: Linking structures to experimental data

### 4. ISSUE DETECTION & FIXING

#### A. Common Issues to Check
1. **Missing Values**
   - Count per column
   - Patterns (always missing together?)
   - Fill strategies

2. **Data Type Mismatches**
   - Numbers stored as strings
   - Dates in various formats
   - Boolean inconsistencies

3. **Duplicates**
   - Full row duplicates
   - Key column duplicates
   - Near-duplicates

4. **Outliers**
   - Statistical outliers
   - Invalid values
   - Data entry errors

#### B. Fix Strategies
```python
# Fix in chunks to save memory
python /claude/csv_helper.py fix file.csv output.csv --fix-missing --fix-types --remove-duplicates

# Manual chunk processing
def process_chunk(chunk):
    # Fix missing values
    chunk.fillna({'col1': 0, 'col2': 'Unknown'}, inplace=True)
    
    # Fix data types
    chunk['price'] = pd.to_numeric(chunk['price'], errors='coerce')
    chunk['date'] = pd.to_datetime(chunk['date'], errors='coerce')
    
    # Remove duplicates
    chunk.drop_duplicates(subset=['id'], inplace=True)
    
    return chunk

# Process file in chunks
pd.read_csv('input.csv', chunksize=10000).pipe(
    lambda chunks: pd.concat(process_chunk(chunk) for chunk in chunks)
).to_csv('output.csv', index=False)
```

### 6. MEMORY-EFFICIENT OPERATIONS

#### A. Never Do This:
```python
# BAD: Loads entire file
df = pd.read_csv('huge_file.csv')
result = df[df['column'] == 'value']

# BAD: Multiple full reads
df1 = pd.read_csv('file.csv')
df2 = pd.read_csv('file.csv')
```

#### B. Always Do This:
```python
# GOOD: Read only what's needed
df = pd.read_csv('file.csv', usecols=['needed_col1', 'needed_col2'])

# GOOD: Process in chunks
results = []
for chunk in pd.read_csv('huge_file.csv', chunksize=10000):
    result = chunk[chunk['column'] == 'value']
    results.append(result)
final_result = pd.concat(results)

# GOOD: Use iterators
with pd.read_csv('file.csv', chunksize=1000) as reader:
    for chunk in reader:
        process(chunk)
```

### 9. DECISION TREE FOR CSV OPERATIONS

1. **File Size Check** → If > 10MB, always use sampling/chunking
2. **Task Type** → Analysis? Use sampling. Transformation? Use chunks.
3. **Column Count** → If > 20 columns, always specify needed columns
4. **Row Count** → If > 100k rows, never load all at once
5. **Memory Available** → Monitor with `free -h` during operations

### 10. ERROR HANDLING

Always wrap CSV operations with proper error handling:

```python
try:
    # Attempt operation
    df = pd.read_csv('file.csv', nrows=1000)
except pd.errors.EmptyDataError:
    print("Empty CSV file")
except pd.errors.ParserError as e:
    print(f"Parse error: {e}")
    # Try with different parameters
    df = pd.read_csv('file.csv', nrows=1000, error_bad_lines=False)
except MemoryError:
    print("Out of memory - use chunking")
    # Fall back to chunk processing
```
